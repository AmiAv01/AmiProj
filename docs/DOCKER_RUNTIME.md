# Docker & Runtime Model

## Overview

The project uses Docker for both development and production. The setup includes:

- **PHP Application** (php:8.4-fpm)
- **Nginx** Web server
- **MySQL** Database
- **Redis** Cache and Queue
- **Queue Worker** (database-driven)
- **Scheduler** (cron alternative)

## Quick Start

### Start all services

```bash
docker-compose up -d
```

### Build fresh images

```bash
docker-compose build --no-cache
```

### Stop services

```bash
docker-compose down
```

### View logs

```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f app
docker-compose logs -f nginx
docker-compose logs -f queue-worker
```

## Service Details

### 1. App (PHP-FPM)
- **Port**: 9000 (internal, not exposed)
- **Workdir**: `/app`
- **Health Check**: `/health/live` every 30s

**Environment Variables:**
```env
APP_ENV=local
APP_DEBUG=true
APP_KEY=<auto-generated>
DB_HOST=mysql
DB_DATABASE=ami_project
QUEUE_CONNECTION=database
REDIS_HOST=redis
```

### 2. Nginx
- **Port**: 80, 443
- **Config**: `docker/nginx/default.conf`
- **Health Check**: GET `/health/live` every 30s

Features:
- SSL ready (mount certificates to `/etc/nginx/certs/`)
- Gzip compression
- Security headers
- Health endpoints bypass logging
- Static file caching

### 3. MySQL
- **Port**: 3306
- **Volume**: `ami_db` (persisted)
- **Health Check**: `mysqladmin ping` every 10s
- **Init Scripts**: `docker/mysql/init/` (runs on first start)

Default credentials:
```env
DB_USERNAME=root
DB_PASSWORD=password
DB_DATABASE=ami_project
```

### 4. Redis
- **Port**: 6379
- **Volume**: `ami_redis` (persisted)
- **Health Check**: `redis-cli ping` every 10s
- **AOF Persistence**: Enabled

Used for:
- Cache
- Queue (alternative to database)
- Sessions (if configured)

### 5. Queue Worker
- **Command**: `php artisan queue:work`
- **Retry Policy**: 3 attempts, 90s timeout
- **Sleep**: 3 seconds between job cycles

Scales horizontally:

```bash
docker-compose up -d --scale queue-worker=3
```

### 6. Scheduler
- **Command**: `php artisan schedule:work`
- **Purpose**: Runs scheduled tasks instead of crontab

## Development Workflow

### Database setup

```bash
# Initial migration
docker-compose exec app php artisan migrate

# Seed data (if seeders exist)
docker-compose exec app php artisan db:seed
```

### Running commands

```bash
# Artisan commands
docker-compose exec app php artisan tinker

# Composer
docker-compose exec app composer require laravel/passport

# Testing
docker-compose exec app php artisan test

# Migrations
docker-compose exec app php artisan make:migration create_table
```

### Accessing services

```bash
# MySQL
docker-compose exec mysql mysql -u root -ppassword ami_project

# Redis
docker-compose exec redis redis-cli

# PHP shell
docker-compose exec app sh
```

## Production Deployment

### Build for production

```bash
# Build image
docker build -t ami:latest .

# Tag for registry
docker tag ami:latest registry.example.com/ami:v1.0.0

# Push to registry
docker push registry.example.com/ami:v1.0.0
```

### Environment for production

```env
APP_ENV=production
APP_DEBUG=false
LOG_CHANNEL=stack
QUEUE_CONNECTION=database
CACHE_DRIVER=redis
SESSION_DRIVER=cookie
SENTRY_LARAVEL_DSN=https://...
```

### Docker Compose for production

```yaml
version: '3.8'
services:
  app:
    image: registry.example.com/ami:v1.0.0
    restart: always
    environment:
      # Use production values from .env.production
      - APP_ENV=production
      - APP_DEBUG=false
    # ... etc
```

### Health checks in production

```bash
# Check liveness
curl http://localhost/api/health/live

# Check readiness
curl http://localhost/api/health/ready
# Returns 503 if database/cache/queue unavailable
```

## Troubleshooting

### Port already in use

```bash
# Find process using port
lsof -i :80      # Nginx
lsof -i :3306    # MySQL
lsof -i :6379    # Redis

# Or use different port in docker-compose
docker-compose -p my_project up
```

### Database connection errors

```bash
# Check MySQL is running
docker-compose logs mysql

# Test connection
docker-compose exec app php artisan tinker
>>> DB::connection()->getPdo()

# Force restart
docker-compose restart mysql
```

### Queue worker not processing jobs

```bash
# Check if worker is running
docker-compose logs queue-worker

# Manually check jobs table
docker-compose exec mysql mysql -u root -ppassword ami_project
> SELECT COUNT(*) FROM jobs;

# Restart worker
docker-compose restart queue-worker
```

### Storage/logs not persisting

Ensure volumes are mounted:
```bash
docker-compose exec app ls -la /app/storage/logs
docker-compose exec app chmod -R 777 storage bootstrap/cache
```

## Advanced: Docker Networking

Services communicate via service name:
- `app` ← PHP-FPM
- `nginx` ← Web server
- `mysql` ← Database (port 3306)
- `redis` ← Cache (port 6379)

All services on `ami_network` bridge network.

To access from outside container:
```bash
# Use localhost with exposed ports
curl http://localhost/api/health/live
mysql -h 127.0.0.1 -u root -ppassword
redis-cli -h 127.0.0.1
```

## CI/CD Integration

### GitHub Actions with Docker

```yaml
build-and-push:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: docker/setup-buildx-action@v2
    - uses: docker/login-action@v2
      with:
        registry: registry.example.com
        username: ${{ secrets.REGISTRY_USERNAME }}
        password: ${{ secrets.REGISTRY_PASSWORD }}
    - uses: docker/build-push-action@v4
      with:
        context: .
        push: true
        tags: registry.example.com/ami:${{ github.sha }}
```

### Kubernetes Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ami-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: ami
  template:
    metadata:
      labels:
        app: ami
    spec:
      containers:
      - name: app
        image: registry.example.com/ami:latest
        livenessProbe:
          httpGet:
            path: /api/health/live
            port: 9000
          initialDelaySeconds: 10
          periodSeconds: 30
        readinessProbe:
          httpGet:
            path: /api/health/ready
            port: 9000
          initialDelaySeconds: 5
          periodSeconds: 10
```
